# TotalAssets

个人总资产计算与追踪系统。自动聚合多源资产数据（股票、基金、加密货币、房产、贵金属、车辆等），实时计算总资产，并以徽章等级直观展示财富量级。

## 核心理念

- **一处查看，全部资产**：无论资产分布在交易所、钱包、银行还是不动产，统一汇总计算。
- **电脑端负责计算，手机端负责查看**：数据抓取、计算全在 PC 端完成，手机端仅做前端展示和用户手动输入，架构清晰，各司其职。
- **徽章等级，一眼知身价**：总资产位数对应徽章等级 A1～An，直观展示财富量级。

## 徽章系统

| 徽章 | 总资产范围 |
|------|-----------|
| A1 | 1～9 元 |
| A2 | 10～99 元 |
| A3 | 100～999 元 |
| A4 | 1,000～9,999 元 |
| A5 | 1 万～9.9 万 |
| A6 | 10 万～99 万 |
| A7 | 100 万～999 万 |
| A8 | 1,000 万～9,999 万 |
| A9 | 1 亿～9.9 亿 |
| A10 | 10 亿～99 亿 |
| ... | 以此类推 |

徽章随总资产实时变化，每次刷新都可能升级（或降级）。

## 资产数据模型

每项资产包含三个核心字段：

| 字段 | 说明 | 示例 |
|------|------|------|
| 名称 | 资产名称 | `TSLA`、`BTC`、`自住房产`、`黄金ETF` |
| 数量 | 持有数量 | `100 股`、`0.5 BTC`、`1 套`、`50 克` |
| 单价 | 单位价格（统一折算为人民币） | `$180 → ¥1296`、`¥500/克` |

总资产 = Σ (数量 × 单价)

## 数据来源

### 数量来源

| 来源 | 说明 | 适用场景 |
|------|------|---------|
| 交易所 API | 通过交易所官方接口获取持仓数量 | 股票、加密货币、基金 |
| 本机 QMT | 读取本机登录的 QMT（迅投量化交易终端）持仓 | A 股持仓 |
| 用户输入/修改 | 手动录入或修改数量 | 房产、贵金属、现金、车辆等 |

### 单价来源

| 来源 | 说明 | 适用场景 |
|------|------|---------|
| 交易所 API | 实时或延迟行情价格 | 股票、ETF、加密货币 |
| 本机 QMT | 通过本机 QMT 获取实时行情 | A 股行情 |
| 公开网络数据 | 爬取/请求公开价格信息 | 黄金价格、二手房成交均价、闲鱼二手车挂单均价、外汇汇率等 |

## 更新机制

- **即时更新**：用户手动触发刷新，立即获取最新数据并重算总资产。
- **定时自动更新**：每日 15:00（默认）自动更新。时区自动检测（优先使用系统时区），也支持用户自行配置时区。

> 选择 15:00 的默认时间是因为 A 股收盘后数据稳定，是一天内资产计算的好时机。

## 系统架构

```
┌─────────────────────────┐     ┌─────────────────────────┐
│       电脑端 (PC)        │     │       手机端 (Mobile)     │
│                         │     │                         │
│  · 数据抓取调度          │     │  · 资产总览仪表盘        │
│  · 交易所 API 调用       │ ◄──► │  · 徽章展示             │
│  · QMT 数据读取          │ API │  · 资产明细列表          │
│  · 公开网络价格采集      │     │  · 手动输入/修改资产      │
│  · 总资产计算引擎        │     │  · 更新历史查看          │
│  · 定时任务调度          │     │                         │
│  · 本地数据存储          │     │                         │
└─────────────────────────┘     └─────────────────────────┘
```

- PC 端负责所有数据获取、计算、存储、定时任务。
- 手机端通过 API 与 PC 端通信，仅做展示和用户输入。
- 手机不在线不影响定时更新（更新在 PC 端执行）。

## 技术栈（规划）

- **后端**：Python（数据抓取、计算、API 服务）
- **前端**：待定（手机端 Web 页面或小程序）
- **数据存储**：SQLite / PostgreSQL
- **定时任务**：APScheduler 或系统 cron
- **API 框架**：FastAPI / Flask

## 快速开始

```bash
# 克隆项目
git clone git@github.com:wzj998/TotalAssets.git
cd TotalAssets

# 创建虚拟环境
python -m venv venv
source venv/bin/activate  # Windows: venv\Scripts\activate

# 安装依赖
pip install -r requirements.txt

# 启动服务
python app.py
```
